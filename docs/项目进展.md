# Text-to-Animation 项目进展记录

> 最后更新：2026-06-29

## 项目概述

根据文字描述生成卡通风格图片，并可将图片合成为 PPT。

- **GitHub 仓库**：https://github.com/NJ-Isidore/Text-to-Animation
- **主分支**：main

---

## 已实现功能

### 1. 图片生成（image_generator.py）

- **文字转图片**：调用通义万相 `wan2.7-image-pro` 模型，根据文字描述生成卡通风格图片
- **卡通风格增强**：自动在 prompt 前添加卡通风格前缀（色彩鲜明、可爱、扁平化设计）
- **批量生成**：支持从文件读取多段描述（空行分隔），逐张生成
- **交互输入模式**：不传文件参数时，逐条输入描述，直接回车结束输入
- **子文件夹命名**：交互时可指定子文件夹名称，图片保存到 `images/<日期>_<名称>/`
- **自动命名**：图片按序号+描述自动命名

### 2. PPT 合成（ppt_builder.py）

- **一键合成**：将 `images/` 子文件夹中的图片自动合成为 16:9 宽屏 PPT
- **智能布局**：图片居中显示，自动适应尺寸
- **标题提取**：从文件名自动提取标题，显示在幻灯片底部
- **多文件夹支持**：自动识别或使用 `--folder` 指定子文件夹

### 3. 运行脚本

| 脚本 | 用途 |
|------|------|
| `install.sh` / `install.bat` | 安装依赖（uv sync） |
| `generate_images.sh` / `generate_images.bat` | 生成图片（支持文件/交互两种模式） |
| `generate_ppt.sh` / `generate_ppt.bat` | 合成 PPT |

- Windows `.bat` 脚本支持**双击运行**，交互式提示输入子文件夹名称和描述
- 也支持命令行传参：`generate_images.bat prompts.txt --name 情侣日常`

## 已实现：人物一致性（参考图功能）

> 实现日期：2026-06-27
> 方案修正日期：2026-06-27（从 OSS 方案迁移为 Base64 直传）

### 功能说明

将参考图以 Base64 编码方式放入 API 的 `content` 数组中，模型会参考输入图片中的人物特征（面部、体型等），同时根据 prompt 描述重新生成衣物、背景、场景等内容，使同一批次生成的图片中人物保持一致。

> **注意**：wan2.7-image-pro API **没有** `ref_img`、`ref_mode`、`ref_strength` 参数（这些属于旧版 wanx-v1 API）。参考图通过 `content` 数组中的 `image` 字段传入，模型根据 prompt 语义决定如何利用输入图片。

### 使用方式

**1. 放置参考图：** 将参考人物图片放入 `reference_images/` 目录

**2. 生成图片：**
```bash
# CLI 指定参考图
scripts/generate_images.sh prompts.txt --ref reference_images/character.png

# 交互模式会提示是否使用参考图
scripts/generate_images.sh
```

### 模块说明

- `image_generator.py` — `call_image_api()` 将参考图 Base64 编码后放入 `content` 数组；`select_reference_image()` 交互选择参考图并返回本地路径；`_find_reference_image()` 自动扫描 `reference_images/` 目录
- `config.py` — 保留 `REFERENCE_IMAGE_DIR` 目录配置，已移除全部 OSS 相关配置
- `main.py` — `--ref` CLI 参数传入本地路径

## 待开发：交互式选择体验优化

> 提出日期：2026-06-28

### 需求 1：子文件夹交互选择

**现状**：`ppt_builder.py` 的 `select_folder()` 在多个文件夹时提示输入数字序号选择，但交互体验简陋——纯文本列表 + 手动输入数字。

**目标**：
- 列出 `images/` 下所有已有子文件夹，展示文件夹名称和图片数量
- 支持**光标上下移动 + 回车**进行选择（类似 `inquirer` / `questionary` 的交互式列表）
- 如果终端支持，也支持**鼠标点击**选择
- 选择后直接使用该文件夹生成 PPT

**涉及文件**：`ppt_builder.py` → `select_folder()` 函数

### 需求 2：参考图交互选择

**现状**：`image_generator.py` 的 `select_reference_images()` 在交互模式下只列出文件名，询问是否全部使用，体验简陋。

**目标**：
- 读取 `reference_images/` 下所有已有参考图
- **优先展示图片预览**（如终端支持则用像素级预览，否则退化为展示文件名列表）
- 支持**光标上下移动 + 回车**进行多选（可勾选多张参考图）
- 如果终端支持，也支持**鼠标点击**选择
- 支持"全选"和"不使用参考图"两个快捷选项

**涉及文件**：`image_generator.py` → `select_reference_images()` 和 `_find_reference_images()` 函数

### 技术方案调研

两个需求的交互方式相似（列表展示 + 光标/点击选择），可选方案：

| 方案 | 说明 | 优劣 |
|------|------|------|
| **questionary** | Python 交互式 CLI 库，支持 checkbox/select，光标移动选择 | ✅ 成熟稳定，API 简洁；❌ 不支持鼠标点击 |
| **inquirer** | 类似 questionary 的 CLI 交互库 | ✅ 功能丰富；❌ 维护不如 questionary 活跃 |
| **textual** | 终端 TUI 框架，支持鼠标点击、丰富组件 | ✅ 可做图片预览和鼠标点击；❌ 较重，学习成本高 |
| **rich + questionary** | rich 展示图片/表格 + questionary 做选择 | ✅ 兼顾美观和轻量；❌ 图片预览受终端限制 |

**推荐方案**：`questionary`（轻量，满足光标移动选择需求）。如果未来需要图片预览和鼠标点击，可升级到 `textual`。

## 已实现：Prompt 增强（解决描述生成图片质量差的问题）

> 提出日期：2026-06-29
> 方案验证日期：2026-06-29（通过官方文档确认方案可行）
> 实现日期：2026-06-30
> 状态：**已实现，待端到端测试**

### 问题背景

当前 `enhance_prompt()` 只是在原始描述前简单拼接卡通风格前缀，对于复杂场景（多人物 + 具体服装 + 空间布局）效果很差。实测发现：

1. **服装属性绑定混乱**：描述"男生短袖短裤、女生穿裙子"，模型把裙子穿到了男生身上
2. **场景元素完全丢失**：描述中的玻璃茶几、墙上电视等物件未出现在图片中
3. **空间关系无法体现**：人物前后纵深关系被模型忽略

### 官方文档验证结论

通过查阅阿里云百炼官方文档（2026-06-29），确认以下关键事实：

#### 1. 两个模型使用同一 API Key，不同 Endpoint

| 用途 | 模型 | Endpoint | 调用方式 |
|------|------|----------|----------|
| **文本生成**（Prompt 增强） | `qwen3.7-max` | `POST /compatible-mode/v1/chat/completions` | OpenAI 兼容格式 |
| **图像生成**（现有功能） | `qwen-image-2.0-pro` | `POST /api/v1/services/aigc/multimodal-generation/generation` | DashScope 原生格式 |

- **Base URL 相同**：`https://{WorkspaceId}.cn-beijing.maas.aliyuncs.com`（当前项目的 WorkspaceId 为 `token-plan`）
- **API Key 相同**：复用 `.env` 中的 `OPENAI_API_KEY`
- **结论：方案完全可行，无冲突**

#### 2. 图像生成 API 的参考图限制

根据 API 返回的错误信息，`qwen-image-2.0-pro`（内部模型名 `qwen-image-2.0-2in1`）对参考图有硬性限制：

| 参考图数量 | 模式 |
|-----------|------|
| **0 张** | 纯文生图（T2I） |
| **1~3 张** | 参考图模式（image-to-image） |
| **超过 3 张** | ❌ 不支持，返回 400 |

**已在代码中修复**：`image_generator.py` 新增 `MAX_REF_IMAGES = 3` 常量，超过 3 张时自动截断并打印 warning。

#### 3. 官方文档链接

- 文本生成（OpenAI 兼容）：https://help.aliyun.com/zh/model-studio/qwen-api-via-openai-chat-completions
- 千问文生图 API：https://help.aliyun.com/zh/model-studio/qwen-image-api
- 文生图 Prompt 指南：https://help.aliyun.com/zh/model-studio/text-to-image-prompt

### 解决方案：两步 Prompt 增强

采用 **方案A（结构化重组）→ 方案B（LLM 改写）** 的流水线，最终输出作为图片生成 API 的描述。

```
用户原始描述 → 方案A 结构化 → 方案B LLM改写 → 图片生成 API
```

#### 方案A：本地结构化重组

在本地对原始 prompt 做关键词检测和模板化重组，确保关键元素不丢失：

- **人物识别**：检测"男生/女生"等关键词，为每个人物建立独立描述段
- **服装隔离**：将每个人的服装描述拆分到独立段落，防止属性串扰
- **场景补全**：检测场景物件（沙发/茶几/电视），缺失时自动补入
- **空间关系**：显式标注前后、左右、上下等方位关系
- **构图指导**：补充画面布局建议（居中/全景/特写等）

**效果**：将一段话拆成结构化的多段描述，降低模型的理解歧义。

#### 方案B：LLM 改写

调用 LLM（`qwen3.7-max`，复用当前 DashScope API）将结构化后的 prompt 改写为更详细、更生动的图像描述：

- 将简短描述扩展为丰富的视觉细节
- 强化空间关系和人物互动的表达
- 保持原始描述的所有关键元素不丢失
- 输出适合图像生成模型理解的详细描述

### 涉及文件变更

| 文件 | 变更内容 |
|------|----------|
| `config.py` | 新增 `TEXT_MODEL = "qwen3.7-max"` 和 `TEXT_API_PATH = "/compatible-mode/v1/chat/completions"`，复用 `API_KEY` 和 `API_BASE_URL` |
| `prompt_enhancer.py`（**新建**） | 核心模块：`restructure_prompt()`（方案A）+ `rewrite_prompt_with_llm()`（方案B）+ `enhance_prompt()`（串联两步） |
| `image_generator.py` | 移除旧的 `enhance_prompt()` 函数，改为 `from text2anim.prompt_enhancer import enhance_prompt`；更新 `generate_single_image()` 中的调用 |

### 架构说明

```
main.py
  └── image_generator.py
        └── prompt_enhancer.py（新建）
              ├── restructure_prompt()      ← 方案A：本地结构化
              ├── rewrite_prompt_with_llm() ← 方案B：LLM改写（qwen3.7-max）
              └── enhance_prompt()          ← 串联：A → B → 输出
```
