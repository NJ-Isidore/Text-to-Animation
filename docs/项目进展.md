# Text-to-Animation 项目进展记录

## 项目概述

根据文字描述生成卡通风格图片，并可将图片合成为 PPT。

---

## 已实现功能

### 1. 图片生成（image_generator.py）

- **文字转图片**：调用通义万相 `wan2.7-image-pro` 模型，根据文字描述生成卡通风格图片
- **卡通风格增强**：自动在 prompt 前添加卡通风格前缀（色彩鲜明、可爱、扁平化设计）
- **批量生成**：支持从文件读取多段描述（空行分隔），逐张生成
- **交互输入模式**：不传文件参数时，可逐条输入描述，空行结束
- **自动命名**：图片按序号+描述自动命名，保存到 `images/<日期>_<名称>/` 子文件夹

### 2. PPT 合成（ppt_builder.py）

- **一键合成**：将 `images/` 子文件夹中的图片自动合成为 16:9 宽屏 PPT
- **智能布局**：图片居中显示，自动适应尺寸
- **标题提取**：从文件名自动提取标题，显示在幻灯片底部
- **多文件夹支持**：自动识别或使用 `--folder` 指定子文件夹

### 3. 运行脚本

| 脚本 | 用途 |
|------|------|
| `install.sh` / `install.bat` | 安装依赖（uv sync） |
| `generate_images.sh` / `generate_images.bat` | 生成图片（支持文件/交互两种模式） |
| `generate_ppt.sh` / `generate_ppt.bat` | 合成 PPT |

Windows 下的 `.bat` 脚本支持**双击运行**，会交互式提示输入参数。

## 待解决问题：人物一致性

### 问题描述

当前每次生成的图片中，情侣角色外貌不一致（发型、服装、面容随机）。需要让同一组图片中的角色保持统一。

### 技术方案调研

通义万相 API 支持 `ref_img`（参考图片）参数，模型会参考该图片的风格或人物特征生成新图。

**API 参数示例：**
```python
payload = {
    "model": "wan2.7-image-pro",
    "input": {
        "messages": [{"role": "user", "content": [{"text": "prompt..."}]}]
    },
    "parameters": {
        "size": "1024*1024",
        "ref_img": "https://example.com/reference.jpg"  # 参考图URL
    }
}
```

### 核心限制

`ref_img` 必须是**公开可访问的 URL**，不支持本地文件路径。

### 可选方案（待决策）

| 方案 | 思路 | 优缺点 |
|------|------|--------|
| **阿里云 OSS 上传** | 将本地参考图上传到 OSS，获取公开 URL | 稳定可靠，但需配置 OSS Bucket |
| **手动提供 URL** | 用户自行上传图片到图床（sm.ms / imgur），交互时粘贴 URL | 简单但每次需手动操作 |
| **文字描述约束** | 不使用 ref_img，在每条 prompt 中加入详细角色外貌描述 | 无需额外配置，但一致性效果有限 |

### 下一步计划

1. 确定参考图方案（建议优先尝试阿里云 OSS 或手动图床 URL）
2. 在 `config.py` 中添加参考图配置
3. 修改 `image_generator.py` 的 `call_image_api()` 支持传入 `ref_img`
4. 交互模式中增加「是否使用参考图」的提示
5. 测试生成效果，评估一致性质量
